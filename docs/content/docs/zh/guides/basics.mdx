首先准备或生成一份 OpenAPI 规范，例如 [petstore.yaml](https://github.com/orval-labs/orval/blob/master/samples/basic/petstore.yaml)。

然后在项目根目录创建 `orval.config.ts`。

## 基础配置

```ts title="orval.config.ts"
import { defineConfig } from 'orval';

export default defineConfig({
  petstore: {
    output: {
      mode: 'single',
      target: './src/petstore.ts',
      schemas: './src/model',
      mock: true,
    },
    input: {
      target: './petstore.yaml',
    },
  },
});
```

## Output 选项

`output` 选项决定生成什么代码，以及生成到哪里。

| 选项 | 说明 |
| --- | --- |
| `mode` | 指定文件生成方式。默认值为 `single`，也就是把所有内容生成到一个文件 |
| `target` | 指定生成文件的目标路径 |
| `schemas` | 指定模型文件的输出路径 |
| `client` | 指定要生成的 HTTP 客户端，例如 `axios`、`react-query`、`vue-query` |
| `mock` | 使用 MSW 生成 mock。参考 [MSW 指南](/zh/docs/guides/msw) |

## Input 选项

`input` 选项描述要导入的规范，以及可选的覆盖配置。

| 选项 | 说明 |
| --- | --- |
| `target` | OpenAPI 规范文件的路径或 URL |
| `override.transformer` | 在生成前转换规范，例如为每个请求追加参数 |
